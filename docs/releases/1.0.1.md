# limine-manager 1.0.1

This maintenance release fixes two false-positive validations found on a real Arch Linux installation.

## Fixes

- Btrfs root sources reported by `findmnt` with an internal subvolume suffix, such as `/dev/mapper/cryptroot[/@]`, are normalized before comparison with the kernel `root=` argument.
- Encrypted roots using mkinitcpio `sd-encrypt` are recognized through one or more `rd.luks.name=UUID=mapper` arguments. Traditional `cryptdevice=UUID=...:mapper` remains supported.

No configuration schema or generated Limine menu format changed.
