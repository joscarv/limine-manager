# limine-manager 1.3.0

Version 1.3.0 completes automatic kernel command-line discovery for supported Arch Linux Btrfs installations.

## Highlights

- A non-empty `/etc/kernel/cmdline` remains an explicit user override.
- If the file is missing or empty, limine-manager generates a canonical command line.
- Unencrypted roots use `root=UUID=<filesystem-uuid>`.
- Encrypted roots discover the dm-crypt mapper, backing block device, and LUKS UUID.
- mkinitcpio `encrypt` generates `cryptdevice=UUID=<luks-uuid>:<mapper>`.
- mkinitcpio `sd-encrypt` generates `rd.luks.name=<luks-uuid>=<mapper>`.
- Btrfs roots include `rootflags=subvol=<root-subvolume>` and `rw`.
- Validation no longer requires encryption arguments for an unencrypted root.

## Compatibility

Configuration schema remains version 1. Existing non-empty kernel command-line files are parsed and validated exactly as before.
