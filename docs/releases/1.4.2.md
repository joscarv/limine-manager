# limine-manager 1.4.2

Version 1.4.2 consolidates encrypted-root discovery for unprivileged status checks.

## Changes

- Resolve dm-crypt backing devices through `/sys/class/block/<dm>/slaves` before invoking privileged tools.
- Keep `cryptsetup status` and `lsblk PKNAME` as fallbacks.
- Populate backing device, LUKS UUID, and PARTUUID consistently when `status` is run without `sudo`.
- Add a regression test for a traditional encrypted Arch Linux installation where `cryptsetup status` is not available to the current user.
