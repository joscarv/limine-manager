# limine-manager 1.1.0

This release adds guarded rollback from a currently booted Snapper root snapshot.

## New commands

- `limine-manager rollback-status`
- `limine-manager rollback-plan`
- `sudo limine-manager rollback`

The rollback flow is intentionally separate from `restore`. `restore` manages backups of `/boot/limine.conf`; `rollback` replaces the main Btrfs root subvolume from the currently booted managed snapshot.

## Safety model

`rollback` refuses to run from the normal root subvolume. For an eligible Snapper root snapshot, it mounts the Btrfs top-level, creates a writable replacement from the booted snapshot, preserves the previous `@` with a unique recovery name, moves the replacement into `@`, verifies the topology, regenerates `limine.conf`, and reports that a reboot is required.

Btrfs does not provide a single atomic transaction for the entire operation, so rollback is implemented as explicit phases with partial recovery after failures.

Configuration schema remains version 1.

## Visual themes

The release also includes built-in clean color presets for the Limine interface:

- Tokyo Night
- Catppuccin
- Nord
- Dracula
- Gruvbox

Use `limine-manager themes` to list presets and `[theme] name = <preset>` to enable one. Presets render Limine-supported interface and terminal options such as `term_palette`, `term_background`, `term_foreground`, and `interface_help_colour`.
