# limine-manager 1.3.2

Version 1.3.2 fixes encrypted-root validation for Archinstall-style installations.

The validator now treats the LUKS UUID as the stable identity of the encrypted root. Mapper names are still checked when available, but a different runtime alias no longer causes a false validation failure when `cryptdevice=` or `rd.luks.name=` references the correct LUKS container.

It also detects the filesystem UUID inside the active dm-crypt mapping so configurations using `root=UUID=...` validate correctly.
