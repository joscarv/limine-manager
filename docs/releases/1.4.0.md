# limine-manager 1.4.0

Version 1.4.0 restructures the application around a centralized system model and turns `status` into the primary installation health report.

## Highlights

- One discovery phase builds a complete `SystemModel`.
- Validation and generation consume the discovered model rather than independently inspecting the host.
- `status` reports root storage, LUKS identity, boot location, kernel/UKI details, Snapper coverage, configuration drift, theme, and backups.
- Validation failures are included directly in `status`.

## Exit codes for status

- `0`: healthy and current.
- `3`: healthy with pending `limine.conf` changes.
- `1`: unhealthy.
